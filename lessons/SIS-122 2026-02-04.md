# SIS-122: Introducción a las Bases de Datos I

Esta sesión introductoria establece las bases del curso, explorando la necesidad técnica de los sistemas de bases de datos, su aplicación en la toma de decisiones (Business Intelligence) y la metodología de evaluación basada en proyectos individuales de diseño institucional.

---

## 1. La Necesidad de las Bases de Datos

La transición de la comunicación verbal a la digital evidencia la necesidad de un sistema de registro persistente y organizado.

### El Ejemplo de la Consulta Manual
Para buscar información sobre un estudiante, el proceso verbal es limitado:
- **Interacción:** "¿Cómo te llamas? ¿Cuántos años tienes? ¿Estado civil? ¿Dirección?"
- **Datos de prueba:** Manuel, 19 años, soltero, residente en una ciudad específica.
- **Identificación Biométrica:** El instructor menciona que si hubiera una base de datos por identificación de rostro, alumnos como Diego, Manuel o Alejandra podrían ser identificados automáticamente sin preguntas.

> [!important] Definición Operativa
> Una base de datos nace de la necesidad de almacenar información organizada de manera que sea consultable por medios digitales cuando el contacto físico o verbal no es posible o eficiente.

---

## 2. Tipos de Bases de Datos

Existen diversos niveles y paradigmas de almacenamiento:

| Tipo | Orientación | Descripción |
| :--- | :--- | :--- |
| **Relacionales** | Estructural y Ordenado | Información organizada en tablas lógicas. Foco del semestre. |
| **No Relacionales** | Objetos / Valores / Vectores | Orientadas a objetos o grandes volúmenes de datos para IA. |

> [!note] Foco del Semestre
> Trabajaremos exclusivamente con **Bases de Datos Relacionales** y el lenguaje **SQL**. Se menciona que SQL Server (Microsoft) es una opción común en servidores, aunque se evaluaron otras como Oracle (descartada por complejidad innecesaria para este nivel).

## 4. Diseño Institucional y Aplicabilidad

El objetivo es diseñar sistemas que registren **transacciones operativas** y permitan la escalabilidad, superando el nivel de una simple "lista de contactos".

### Áreas de Aplicabilidad Sugeridas:
- **Comercial:** Stock, ventas, gastos, facturas.
- **Tecnología avanzada:** **Bitcoin**, **Blockchain** (registro de transacciones/pagos) y **Data Mining**.
- **Comparativa:** No es lo mismo una base de datos para un banco que una para minería de datos o registros de transacciones en cripto.

---

## 5. Business Intelligence e Investigación de Datos

Las bases de datos permiten extraer información estratégica (Business Intelligence) y realizar investigación científica.

> [!example] Caso de Estudio: La Farmacia y la Salud
> **Análisis Comercial:** Identificar el "Top 10" de productos más vendidos por temporada (verano, invierno, otoño). Si en verano hay alta demanda de un producto, la decisión es aumentar el stock.
> **Investigación Científica:** Determinar la cantidad de pacientes que consumen paracetamol en épocas de lluvia (picos en febrero). Investigar si existe relación entre el consumo prolongado de paracetamol y el incremento de casos de **cáncer vesicular** mediante el cruce de registros históricos.

## 6. Componentes Básicos y Estructura (Excel vs. SQL)

Conceptualmente, una base de datos relacional se compone de **Tablas**.

- **Columnas (Campos/Atributos):** Nombres claros como Código, Descripción, Precio, Stock.
- **Filas (Registros/Tuplas):** Los datos específicos.

### El Problema de la Redundancia en una Sola Tabla
El instructor pone el ejemplo de **Juan Pérez**:
- En una tabla única, tendrías que repetir: "Juan Pérez - Matemáticas", "Juan Pérez - Química", "Juan Pérez - Lenguaje".
- Esto genera registros duplicados y dificulta la gestión manual cuando hay **millones de registros**. SQL resuelve esto mediante un diseño relacional científico.
